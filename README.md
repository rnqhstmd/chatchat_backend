## Develop Convention
#### Commit
| **커밋 유형** | **의미** |
| --- | --- |
| **Feat** | 새로운 기능 추가, 기존의 기능을 요구 사항에 맞춰 수정 |
| **Fix** | 버그 수정 |
| **Build** | 빌드 관련 수정, 모듈 설치 또는 삭제 등 |
| **Style** | 코드 스타일 혹은 포맷 등에 관한 커밋 |
| **Refactor** | 기능의 변화가 아닌 코드 리팩토링 |
| **Chore** | 패키지 매니저 수정, 그 외 자잘한 수정 ex) .gitignore |
| **Docs** | 문서 수정 (ex. README.md) |
